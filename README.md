## 프로젝트 개요
본 프로젝트는 **'챗봇을 활용한 인수인계 플랫폼'** 을 위해 개발된 RAG 기반 챗봇의 백엔드 서버에 Flutter의 WebViewWidget을 연동한 시스템입니다.

다수의 사용자와 그룹을 안정적으로 지원하기 위해 **멀티테넌시(Multi-tenancy) 아키텍처**를 적용하였으며, 클라이언트 환경(웹/모바일)에 구애받지 않는 유연한 **RESTful API**를 제공합니다.<br>
모바일 앱 환경에서의 작동 및 서버 연동 메커니즘을 검증하기 위해 **Flutter WebView** 기반의 통신 테스트를 성공적으로 완료했습니다.

---
## 주요 구현 기능
### 1. 멀티테넌시 구조의 대화 세션 관리 (캡스톤 담당 파트)
* **Space ID & User ID 기반 데이터 격리:** 사용자와 작업 공간을 기준으로 데이터를 분리하여, 사용자의 각각의 스페이스에 따른 독립적이고 챗봇 대화 세션을 보장합니다.
* **RESTful API 설계:** 프론트엔드와의 원활한 데이터 통신을 위해 REST 규격에 맞춰 챗봇 질의응답을 설계 및 구현했습니다.
* **사용자 대화 로그(History) 관리 및 API 설계**: 과거 질의응답 내역을 DB에 안전하게 저장하고 불러오는 대화 이력 조회 API를 구현했습니다. 이는 일회성 대화를 넘어 연속적인 세션을 유지하게 하며, 사용자 로그 수집 및 서비스 개선을 위한 데이터 분석의 기반이 됩니다.

### 2. RAG 기반 검색 및 파일 관리 시스템 (캡스톤 담당 파트)
* **실시간 문서 임베딩(Ingestion) 파이프라인 연동**: 인수인계 문서가 업로드될 때, 해당 데이터가 챗봇의 도메인 지식으로 즉각 반영(Ingest) 될 수 있도록 문서 벡터화 및 학습 파이프라인을 연결했습니다.
* **LLM 연동 데이터 처리:** 업로드된 문서를 기반으로 사용자 질문에 정확한 답변을 생성하는 RAG 아키텍처의 백엔드 데이터 흐름을 구축했습니다.

### 3. Flutter WebView 연동
* **모바일 이식성 검증:** 구현된 Spring Boot 기반의 챗봇 웹 화면을 WebViewWidget를 활용해 실제 스마트폰 기기 화면에 렌더링 했습니다.
* **앱-서버 데이터 통신 확인:** 모바일 WebView 환경에서도 Spring 서버의 REST API와 정상적으로 데이터를 주고받으며 작동하는 것을 검증했습니다.
